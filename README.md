1.Что делает fetch?

Команда fetch — это современный API в JavaScript, предназначенный для выполнения сетевых запросов (чаще всего HTTP/HTTPS) к удаленным ресурсам, таким как внешние API. Его основное назначение — асинхронная загрузка данных (JSON, изображений и т.д.), которая происходит без блокировки основного потока выполнения браузера. В результате fetch всегда возвращает объект Promise (Обещание), который разрешается в объект Response после получения заголовков. Для извлечения фактического тела данных (например, JSON) из объекта Response требуются дополнительные асинхронные шаги, такие как вызов метода response.json().                               

2.Зачем нужны async/await?

Синтаксис async/await используется для значительного упрощения работы с Промисами (Promises), делая асинхронный код визуально похожим на синхронный, что значительно повышает его читаемость и удобство отладки. Ключевое слово async объявляет функцию асинхронной, гарантируя, что она всегда вернет Promise. Ключевое слово await можно использовать только внутри такой функции; оно заставляет выполнение функции приостановиться до тех пор, пока Promise не будет выполнен (успешно или с ошибкой), а затем возвращает его результат. Это позволяет обрабатывать последовательные асинхронные операции и ошибки с помощью знакомых блоков try...catch.

3.Как работает localStorage?

localStorage — это механизм веб-хранилища, который позволяет веб-приложениям хранить данные (пары "ключ/значение") в браузере пользователя. Его ключевая особенность — постоянство: данные, сохраненные в localStorage, сохраняются бессрочно и остаются доступными даже после закрытия браузера или перезагрузки компьютера. Все данные хранятся только в виде строк. В нашей работе localStorage был использован для сохранения выбора цветовой темы (dark-theme/light-theme), чтобы пользователь при повторном посещении сайта сразу видел ранее выбранный интерфейс.

4.Где помог ИИ, а где пришлось разбираться вручную?

При выполнении лабораторной работы ИИ (Gemini) оказал значительную помощь, сгенерировав и адаптировав чистую асинхронную функцию loadReview() с использованием fetch и логики async/await для загрузки одной цитаты. Он также быстро исправил критическую ошибку с видимостью текста отзыва в светлой теме, предоставив нужные стили в CSS. Однако пришлось разбираться вручную с интеграцией функции искусственной задержки (delay(500)) для визуализации статуса "Загрузка...", чтобы асинхронный процесс был заметен. Кроме того, ручная работа включала финальное тестирование и отладку интеграции всех блоков (тема, отзывы, галерея, аккордеон) для обеспечения их бесперебойной совместной работы.
